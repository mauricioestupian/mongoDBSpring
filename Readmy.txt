- controller → Aquí van tus controladores REST (UsuarioController) que exponen los endpoints.
- service → Contiene la lógica de negocio. La interfaz (UsuarioService) y su implementación (UsuarioServiceImpl).
- repository → Interfaces que extienden MongoRepository o usan @Query para acceder a la base de datos (MongoDB).
- mapper → Clases que convierten entre entidades y DTOs (UsuarioMapper, UsuarioMapperImpl).
- dto → Objetos de transferencia de datos, usados para exponer información al cliente sin mostrar toda la entidad.
- model → Tus entidades que representan documentos en MongoDB (Usuario, Documento, etc.).
- exception → Excepciones personalizadas y el GlobalExceptionHandler para manejar errores de forma emas explicita y con buensa practicas.
- security → utilidades y componentes de seguridad (como PasswordEncoderUtil, clases de autenticación, filtros personalizados) - lógica de seguridad.
- config →  - es parte de la configuración de Spring Security(SecurityFilterChain) - configuración de Spring

uso de MAPSTRUCT
para mejor uso de mapper
👉 MapStruct es una librería de Java que convierte automáticamente objetos.
DTO  →  Entity
Entity → DTO

✔ Genera la implementación automáticamente
✔ En tiempo de compilación (rápido 🔥)
